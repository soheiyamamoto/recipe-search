# 差分仕様: 材料表示機能(カード材料タグのクリック絞り込み)

- 種別: 差分仕様(spec-writer)/ 親仕様: [spec_recipe-search.md](spec_recipe-search.md)
- 追加日: 2026-07-05
- 対象実装: [../src/index.html](../src/index.html)(RECIPES 区間外の CSS / JS)

## 1. 背景・目的

各レシピカード下部の材料タグ(`材料: きゅうり` 等)は従来 **表示のみ** で操作できなかった。
上部のクイックタグ(頻出上位14件)だけが絞り込みに使え、カード内のタグは死んでいた。
本機能では **カード材料タグをクリック/タップで絞り込みできる** ようにし、
上部クイックタグと一貫した操作性を与える。

## 2. 変更内容

### 2.1 カード材料タグのボタン化(表示)
- `renderCards()` のタグを `<span class="tag">` から `<button type="button" class="tag" data-tag="…">`
  に変更(見た目は踏襲。`cursor:pointer`・ホバー・キーボードフォーカス可)。
- 表示文言は従来どおり `材料: {タグ}`。`data-tag` には接頭辞なしの素のタグ名を格納。

### 2.2 タグ絞り込みの挙動(クイックタグと共通化)
- クリック時の処理を `applyTagFilter(tag)` に共通化し、クイックタグ・カードタグ双方で使用:
  - 検索欄にタグ名を入れて絞り込む(既存クイックタグと同じ)。
  - 同じタグを再クリックすると解除(トグル)。
  - 対応するクイックタグ・カードタグを `.active` でハイライト。
  - クリック後、画面先頭へ即時スクロール(検索欄・件数・季節タブが見える)。
    全件時はページ高が約87,000pxと非常に長く、smooth では戻りが遅いため instant を採用。

### 2.3 複合タグの一致漏れ修正(データ整合)
- 表示タグには `牛肉`『米・ご飯』『味噌汁・汁物』等、`materials`(表記ゆれ配列)に
  部分文字列として存在しない語がある。従来の「検索文字列 ⊂ materials/タイトル」判定では
  これらのタグでクリックしても **クリックした当該カード自身が一致せず消える** 不具合があった
  (既存クイックタグにも潜在)。
- 対策: カードに `data-tags`(表示タグを `|` 連結・小文字化)を付与し、
  `filterRecipes()` の一致判定に **タグ一致** を追加:
  `matchesSearch = 空 || materials一致 || タイトル一致 || タグ一致`。
  これによりタグ経由の絞り込みが常に当該カードを含む。自由入力検索も表示タグにヒットするよう改善。

## 3. 受け入れ確認項目(6点)

1. カード下部の各材料タグがクリック/タップ可能(ボタン、`cursor:pointer`、Tabフォーカス可)。
2. タグをクリックすると一覧が当該材料で絞り込まれ件数表示が更新。**クリックしたカードは必ず残る**
   (`牛肉`『米・ご飯』等の複合タグでも一致漏れなし)。
3. 上部クイックタグと挙動一致(検索欄にタグ名反映 / 同タグ再クリックで解除 / 対応タグがハイライト)。
4. タグクリック時に画面が先頭へスクロールし、検索欄・件数・季節タブが視認できる。
5. 季節タブと併用可能(季節絞り込み中にタグをクリックしても季節条件が維持され AND で絞り込み)。
6. ✕(クリア)で材料フィルタが解除され現季節の全件に戻る。既存の検索/0件表示/全318件表示に回帰なし。

## 4. 非対象(今回やらないこと)
- 材料インデックス(全材料一覧・件数付き)パネルの追加。
- カード表示タグの上限(最大3件)拡張。
- 材料辞書 MATERIAL_DICT / 季節辞書の変更(親仕様 spec_recipe-search.md 側の管轄)。
